常见字符集及编码

发表于 2021-04-18 | 分类于 Java基础

基本概念

  • 字符 Character
  • 字符集 Character set
  • 字符编码 CHaracter encoding

    字符集和字符编码一般都是成对出现的，如ASCII、IOS-8859-1、GB2312、GBK，都是即表示了字符集又表示了对应的字符编码。
    Unicode比较特殊，有多重字符编码（UTF-8，UTF16）。

常见字符集及编码

1. ASCII

全称

American Standard code for information interchange

字符集范围

128个字符： 33个控制字符（大部分废弃） 95个可现实字符

编码格式

单字节，
一个8位二进制数，最高位是0 ，可用为7位：三位 * 四位 = 8 * 16 =128    
"0"-48, "a"-97, "A"-65

兼容

Unicode GBXXXX UTF-8 都兼容ASCII
EASCII是对ASCII的扩展 7位变8位 共256个字符
多了表格符号 计算符号，希腊字母与特殊的拉丁符号

2. GB2312

全称

GB 2312或GB 2312-80
简体中文字符集,又称GB0，

字符集范围

收录6763个汉字   
不能覆盖人名古汉语的罕用字和繁体字

编码格式

两个字节
每个汉字由2个字节表示 高位字节+低位字节
区位码 
 - 01-09区 特殊符号
 - 16-55区 一级汉字 拼音排序（3755个）
 - 56-87区 二级汉字 不收笔画排序（3008个）
>例如 “啊”字位于16区01位
 占用的码位72* 94=6768 有五个空位

兼容

GB 2312兼容ASCII

3. GBK

全称

汉字内码扩展规范GBK（扩展的Kuo的K）
英文全称Chinese Internal Code Extension Specification

字符集范围

拥有23940码位
共收入21886个汉字（包括部首和构件）21003个和图形符号883个

编码格式

双字节

兼容

向下兼容GB 2312-80，但不支持韩国字。
与此相比Unicode支持。

4. GB 18030

全称

国家标准GB 18030-2005《信息技术中文编码字符集》

字符集范围

拥有161万个字元
收录汉字70144个
支持少数民族文字
收录繁体汉字、以及日韩文字

编码格式

多字节
同UTF-8，每个字可以由1个、2个、4个字节组成

兼容

与GBK**基本**兼容 
支持Unicode全部统一汉字

5. Unicode

全称

万国码、国际码、统一码、单一码，业绩标准。

字符集范围

超过10万字符

编码格式

2个字节，UCS-2，16位编码空间，2^16=65536
为避免空间浪费，有多种实现，Unicode转换格式（Unicode Transformation Format简称UTF）
UTF-8，UTF-16，UTF-32都是数字转换到程序的编码方案

兼容

兼容ASCII

6. UTF-8

全称

8-bit Unicode Transformation Format,Unicode的可变长度编码，是一种前缀码

字符集范围

1字节：128个US-ASCII
2字节：拉丁文、希腊文...
3字节：大部分汉字
4-6字节：极少使用的辅助平面的字符4-6位

编码格式

使用1-6个字节编码，
  • 1字节的字符，以0开头的，0xxx xxxx，后面7个有效位，2的7次方，最多可以表示128种字符。
  • 2字节的字符，110x xxxx 10xx xxxx，数一数，11个x，所以是2的11次方，2的10次方是1024，11次方就是2048，很不幸，只能表示2048种字符，而我们的常用汉字就有3000多个，看来在这一区是放不下了，只好挪到3字节。
  • 3字节的字符，1110 xxxx 10xx xxxx 10xx xxxx，数一数，16个x，2的16次方，最多可以表示65536个字符，所以我们的汉字就放在这一区，所以在UTF-8方案里我们的汉字都是以3个字节表示的。
有效位 最大位 Byte 1 Byte2 Byte 3 Byte 4
7 U+007F 0xxxxxxx
11 U+07FF 110xxxxx 10xxxxxx
16 U+FFFF 110xxxxx 10xxxxxx 10xxxxxx
21 U+1FFFF 110xxxxx 10xxxxxx 10xxxxxx 10xxxxxx

UTF-8和UTF-16区别

那么UTF-8的8是从哪儿来的呢？它的意思就是说我们以2的8次方为一个字节，为一个最小单元。
那么如果我们以2的16次方为一个最小单元，这就变成了UTF-16，它的规则和UTF-8相同，
唯一不同的是它最小也要用16个2进制位表示一个字符，而16个2进制位直接可以表示65536种字符，
所以在UTF-16方案里，我们汉字直接就可以如英文一样被堂而皇之地放在第1区了，也就是说，
和英文具有同等的身份，都占用16个2进制位，也就相当于UTF-8里的2字节哦，看，这样一来，
如果我们用UTF-16来存储英文的话，会造成浪费，因为英文在UTF-8里只占1字节，
而在UTF-16里要占2字节，但是如果我们用UTF-16来存储中文的话，不但不浪费，
反而还节省了呢！因为我们的中文在UTF-8里要占用3字节，而在UTF-16里只占用2字节，
节省了33%之多呢！
